⏺ Here is a complete breakdown of the MyQuizApp UI structure and what needs
  improvement:

  ---
  Current UI Structure

  Component Architecture

  Three files make up the entire UI:

  ┌────────────────────────────┬─────────────────────────────────────────────┐
  │            File            │                    Role                     │
  ├────────────────────────────┼─────────────────────────────────────────────┤
  │ App.js                     │ Entry point, navigation setup, quiz data    │
  ├────────────────────────────┼─────────────────────────────────────────────┤
  │ src/components/Question.js │ Single question display + answer            │
  │                            │ interaction                                 │
  ├────────────────────────────┼─────────────────────────────────────────────┤
  │ src/components/Summary.js  │ Results screen after quiz completion        │
  └────────────────────────────┴─────────────────────────────────────────────┘

  The navigation is a simple 2-screen stack: Question → Summary. Each question
  gets push()-ed onto the stack, so the back button works per-question by
  calling goBack().

  ---
  Styling Approach — Inconsistent and Fragmented

  - No central theme file, no design tokens, no color constants. Everything is
  scattered.
  - Summary.js uses StyleSheet.create() (lines 68–99) — the correct React Native
   pattern.
  - Question.js uses only inline styles — { flex: 1, padding: 20 }, { fontSize: 
  18 }, etc.
  - These two components were styled by different people or at different times,
  with no shared foundation.

  Component library fragmentation:
  - ButtonGroup from react-native-elements — answer choices
  - Button from native React Native — the Next/Finish action
  - Icon from react-native-vector-icons — back arrow
  - No wrapper layer, no theming provider, no unified visual contract

  ---
  Layout Patterns

  - Layouts are entirely flex: 1 column stacks with raw pixel padding/margins
  - Spacing values scattered: 3, 15, 20 px — no spacing scale
  - No responsive sizing (Dimensions, Platform.select, %-based values not used)
  - ScrollView in Summary handles overflow, but Question screen has no scroll —
  long questions or many choices could overflow

  ---
  Color Palette

  ┌────────────────────────┬─────────────────────────────────────────────┐
  │         Color          │                    Usage                    │
  ├────────────────────────┼─────────────────────────────────────────────┤
  │ System default (black) │ All body text                               │
  ├────────────────────────┼─────────────────────────────────────────────┤
  │ 'green'                │ Correct answer feedback in Summary          │
  ├────────────────────────┼─────────────────────────────────────────────┤
  │ 'red'                  │ Incorrect answer feedback in Summary        │
  ├────────────────────────┼─────────────────────────────────────────────┤
  │ '#FFFFFF'              │ Splash screen background (Android XML only) │
  └────────────────────────┴─────────────────────────────────────────────┘

  No primary brand color, no accent, no surface/background system. Named colors
  like 'green' and 'red' are not standardized — they render differently across
  devices and don't match any defined palette.

  ---
  Typography

  ┌──────┬─────────────────────────────────────────────────────────┐
  │ Size │                          Usage                          │
  ├──────┼─────────────────────────────────────────────────────────┤
  │ 18px │ Question prompt (Question.js), choice text (Summary.js) │
  ├──────┼─────────────────────────────────────────────────────────┤
  │ 20px │ Question text in Summary review list                    │
  ├──────┼─────────────────────────────────────────────────────────┤
  │ 24px │ Total score heading                                     │
  └──────┴─────────────────────────────────────────────────────────┘

  - No custom fonts — system defaults only
  - No defined type scale or heading/body hierarchy
  - fontWeight: 'bold' used in two places (correct answers, total score) but no
  weight system
  - textDecorationLine: 'line-through' used for struck-through wrong answers —
  visually heavy but not designed

  ---
  Visual Weaknesses

  1. No design system at all
  Zero shared constants, no colors.js, no theme.js, no spacing scale. Every
  value is a magic number defined inline.

  2. Styling methodology mismatch
  Question.js (inline) vs Summary.js (StyleSheet) — a new contributor has no
  obvious pattern to follow.

  3. Native Button is unstyled
  The "Next Question" / "Finish Quiz" button uses React Native's bare <Button> —
   which renders as a blue text label on iOS and a flat OS-native button on
  Android. It cannot be styled for color, padding, shape, or border radius.

  4. No visual feedback on answer selection
  Answer choices use ButtonGroup but there is no visible "selected" state
  customization — the default react-native-elements styling is unchanged and
  visually dated.

  5. Score display is plain text
  The Summary screen shows "You got X out of Y correct" as unstyled body text
  alongside a raw list of questions and answers with no visual hierarchy.

  6. No empty/loading/error states
  The app assumes data is always present and valid — no skeleton, no loading
  indicator, no fallback.

  7. Hard-coded layout values throughout
  Over 20 instances of magic pixel numbers. Changing the spacing of the app
  requires editing multiple unrelated files.

  8. Navigation header is bare
  Only headerTitleAlign: 'center' is set. No title text shown on the Question
  screen, no background color, no custom font — it inherits OS defaults.

  ---
  Modernization Priorities

  High impact / foundational:
  1. Create a src/theme.js with color constants, a spacing scale, and font size
  tokens
  2. Replace the native Button with a custom styled TouchableOpacity or
  Pressable with proper border radius, background color, and touch feedback
  3. Migrate all inline styles in Question.js to StyleSheet.create()

  Visual polish:
  4. Replace 'green'/'red' with proper hex values from the theme
  5. Add a custom font (e.g., Inter via Expo Google Fonts)
  6. Style the navigation header (background, title, font)

  UX improvements:
  7. Add a visible selected-state style to answer choices (border highlight,
  background tint)
  8. Redesign the Summary screen — score as a large hero number, per-question
  results as cards with clear correct/incorrect iconography
  9. Add a "Restart Quiz" button on Summary

  Architecture:
  10. Consolidate component library — either go all-in on react-native-elements
  (and theme it) or replace it with react-native-paper or a custom component set
   — the current mix is a maintenance liability
